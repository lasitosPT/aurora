Why aurora

Motion and polish, without the lock-in.

Every component is a standard custom element — self-contained, themeable, and accessible by default.

🧩

Framework-agnostic

Native custom elements work the same in React, Vue, Svelte, or a raw HTML file.

✨

GSAP-powered

Fluid, interruptible animations built on the industry-standard motion engine.

🌐

Three.js, opt-in

3D lives on a separate entry, so the core never ships the heavy renderer.

📝

Form-associated

Inputs, switches, and sliders submit natively via ElementInternals.

🎨

Themeable

Style everything with CSS custom properties that pierce the shadow boundary.

♿

Accessible

ARIA roles, keyboard support, and prefers-reduced-motion throughout.

Install

Up and running in two lines.

Install from npm, then import once — components register themselves.

npm

The recommended path for bundled apps.

npm install aurora
// registers the core components
import 'aurora'

// registers <aurora-scene> (pulls in Three.js only here)
import 'aurora/three'

Use anywhere

They are custom elements, so the markup is identical across frameworks.

// React — no wrapper needed
export function Hero() {
  return (
    <aurora-text stagger="0.06">
      <h1>Motion, built in.</h1>
    </aurora-text>
  )
}
Theming

One accent, everywhere.

Set CSS custom properties on :root or any ancestor — they cross the shadow boundary into every component.

Set your tokens

Sensible fallbacks mean you only override what you need.

:root {
  --aurora-accent: #6d5cff;
  --aurora-accent-hover: #5a49e0;
  --aurora-surface: #16161f;
  --aurora-fg: #ececf2;
  --aurora-border: rgba(255, 255, 255, 0.14);
}

Common variables

A few of the tokens the components read.

Variable Purpose
--aurora-accent Primary accent
--aurora-surface Modal / panel background
--aurora-border Dividers & outlines
--aurora-muted Secondary text
